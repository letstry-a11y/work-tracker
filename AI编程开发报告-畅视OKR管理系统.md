## —— 以「畅视OKR管理系统」为例

## 一、项目概述

### 1.1 项目背景

为提升团队目标管理和工作追踪效率，需要一套轻量化、可快速部署的OKR管理系统。传统开发模式下，此类全栈应用的开发周期通常需要 **2-3名开发人员、4-6周** 的时间。本项目采用 **AI辅助编程（Claude Code）** 的方式，由1人在 **约1周内** 完成了从需求分析到生产部署的全流程。

### 1.2 项目成果概览

| 指标 | 数据 |
|------|------|
| 总代码量 | **7,590 行**（不含依赖） |
| 数据库表 | **10 张**核心业务表 |
| API 接口 | **60 个** RESTful 端点 |
| 前端模块 | **15 个** JS 模块 |
| UI 页面 | **8 个**主功能页 + **16 个**弹窗组件 |
| 数据库索引 | **21 个**性能索引 |
| 开发周期 | 2026.04.02 - 2026.04.08（**约 7 天**） |
| 开发人员 | **1 人**（AI辅助） |

## 二、需求输入

### 2.1 核心需求

系统需要支持团队日常OKR目标管理，涵盖以下核心业务场景：

```
1. OKR目标管理      → 整体目标 / 个人目标 / 审批流程
2. KR任务管理        → 任务分配 / 状态追踪 / 优先级管理
3. 每日工作填报      → 工时记录 / 工作内容 / 进度追踪
4. 交付物管理        → 文件上传 / 审核确认
5. 审核记录          → 统一审核（任务/交付物/目标）
6. 部门管理          → 部门架构 / 成员管理 / 负责人设置
7. 员工档案          → 员工信息 / 用户账号 / 角色权限
8. 数据仪表盘        → 统计概览 / 周报热力图 / 逾期预警
```

### 2.2 角色权限体系

| 角色 | 权限范围 |
|------|---------|
| **管理员（admin）** | 全部管理权限，包括用户管理、目标审批、部门管理 |
| **部门负责人（dept_leader）** | 部门内审核、成员管理、目标审批 |
| **普通员工（employee）** | 个人OKR管理、每日填报、交付物提交 |

### 2.3 非功能需求

- 轻量部署：支持 Docker 一键部署
- 局域网访问：支持团队内网使用
- 数据安全：密码加密存储、会话管理、防暴力破解

## 三、架构设计

### 3.1 技术选型

| 层次 | 技术方案 | 选型理由 |
|------|---------|---------|
| **前端** | HTML5 + CSS3 + ES6+ Modules | 零框架依赖，轻量快速，无需构建工具 |
| **后端** | Node.js + Express 4.18 | 成熟稳定，生态丰富，开发效率高 |
| **数据库** | SQLite（sql.js） | 嵌入式数据库，免运维，文件级部署 |
| **认证** | bcryptjs + Session Token | 工业级密码哈希，7天会话有效期 |
| **安全** | express-rate-limit + multer | 接口限流 + 文件上传安全过滤 |
| **部署** | Docker + docker-compose | 容器化一键部署，环境一致性 |

### 4.4 安全设计

| 安全措施 | 实现方式 |
|---------|---------|
| 密码安全 | bcrypt（12轮），自动从SHA256迁移到bcrypt |
| 会话管理 | 随机Token（crypto.randomBytes(32)），7天过期 |
| 接口限流 | express-rate-limit 防暴力破解 |
| XSS防护 | 前端统一 esc() 转义，后端参数化查询 |
| 文件安全 | 上传大小限制100MB，危险扩展名黑名单(.exe/.bat/.sh等) |
| 权限控制 | 三级角色（admin/dept_leader/employee），中间件鉴权 |

```

```
